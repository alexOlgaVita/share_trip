1. Проект реализует сервис для совместных поездок ShareTrip. 
2. Используемые инструменты (fmt, linter, тесты).
   Работа с PostgreSQL через PGX.
   Реализация HTTP-сервера на Fiber.
   gRPC и Proto.
   Архитектура: SOLID, KISS, DRY, GRASP.
   Первый проект: сервис для совместных поездок ShareTrip.
3. Краткое описание проекта: Есть 2 вида участников: Водитель и Пассажир.
   Водитель публикует поездку. Пассажир выбирает поездку, отправляет запрос. Водитель может подтвердить или отклоноить заявку Пассажира.
   Система организует согласованность действий участников и всего процесса в целом.
